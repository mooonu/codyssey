# STEP 8: 텍스트 파일과 이진 파일의 차이점

## 텍스트 파일

사람이 읽을 수 있는 문자들로 이루어진 파일이다.
메모장 같은 프로그램으로 열면 저장된 내용을 바로 확인할 수 있다.

**장점**
- 어떤 운영체제나 프로그램에서도 열고 수정할 수 있다.
- 내용을 사람이 직접 눈으로 확인하고 수정하기 쉽다.
- CSV, JSON처럼 다른 시스템과 데이터를 주고받을 때 호환성이 높다.

**단점**
- 숫자나 복잡한 데이터를 저장할 때 문자로 변환해야 해서 파일 크기가 커진다.
- 데이터를 읽을 때 문자를 다시 숫자나 객체로 변환하는 과정이 필요해서 느리다.

**이 프로젝트에서의 예**
- `Mars_Base_Inventory_List.csv` — 메모장으로 열면 물질 이름과 수치를 바로 볼 수 있다.
- `Mars_Base_Inventory_danger.csv` — 위험 물질 목록을 텍스트로 저장한 파일이다.

## 이진 파일

컴퓨터가 처리하는 0과 1(바이트) 형태 그대로 저장된 파일이다.
메모장으로 열면 알 수 없는 문자들이 표시되어 사람이 내용을 읽기 어렵다.

**장점**
- 데이터를 변환 없이 그대로 저장하므로 파일 크기가 작고 읽고 쓰는 속도가 빠르다.
- Python의 리스트, 딕셔너리 같은 객체를 그 형태 그대로 저장하고 복원할 수 있다.

**단점**
- 저장할 때 사용한 프로그램이나 버전이 다르면 파일을 읽지 못할 수 있다.
- 내용을 사람이 직접 확인하거나 수정하기 어렵다.

**이 프로젝트에서의 예**
- `Mars_Base_Inventory_List.bin` — Python의 `pickle`로 저장한 파일로, `pickle.load()`로만 읽을 수 있다.

---

## 비교 요약

| 구분 | 텍스트 파일 | 이진 파일 |
|------|------------|----------|
| 사람이 읽을 수 있는가 | 읽을 수 있다 | 읽기 어렵다 |
| 파일 크기 | 상대적으로 크다 | 상대적으로 작다 |
| 읽기/쓰기 속도 | 상대적으로 느리다 | 상대적으로 빠르다 |
| 다른 프로그램과 호환 | 호환성이 높다 | 저장한 프로그램에 의존한다 |
| 이 프로젝트 예시 | `.csv` | `.bin` |

---

## 실제 파일 크기 비교 (이 프로젝트 기준)

프로그램 실행 후 두 파일의 크기를 비교하면 이진 파일의 특성을 직접 확인할 수 있다.

```
Mars_Base_Inventory_List.csv  ← 텍스트 파일
Mars_Base_Inventory_List.bin  ← 이진 파일 (pickle)
```

pickle은 Python 객체를 그대로 직렬화하므로 이 프로젝트에서는 .bin이 .csv보다
크기가 클 수도 있다. 이는 pickle 포맷이 객체 타입 정보도 함께 저장하기 때문이다.
일반적으로 이진 파일이 작은 이유는 정수·실수를 문자열 대신 바이트로 저장하기 때문이며,
pickle처럼 메타데이터가 포함된 이진 포맷은 예외가 될 수 있다.
